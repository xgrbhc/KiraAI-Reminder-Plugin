# 智能提醒插件 (Reminder Plugin) v1.0

Kira 官方/社区生态的一款功能强大、高稳定性的全能提醒管理插件。

## ✨ 核心特性

- **📅 多维时间调度**：支持精准定时、周期重复（每天/周/月/年）、间隔循环（每 N 分钟）。
- **🎲 动态随机提醒**：支持在指定时间段内触发 N 次随机时间点的提醒，让交互更自然。
- **🚀 极简快捷指令**：提供类似命令行 (CLI) 的高效指令交互（如 `/r add`, `/r rm` 等），无需等待 LLM 思考即可极速响应处理。
- **🌐 独立 Web 大屏看板**：内置跨域微服务（默认端口 `8080`），提供全局/个人多视角待办可视化白板，支持局域网外网监控。
- **🛡️ 权限与防误删保护**：全局超级管理员可穿梭透视所有用户数据；LLM 尝试删除被标记为重要（⭐）的提醒时强制向客户端发起二次确认。
- **💾 工业级稳定性**：基于分布式概念的并发安全锁写入机制、后台长驻守护线程（调度器健康检查与自动重启）、触发自动异常重试与数据自动纠错清理。

## 📦 安装与配置

1. 在 KiraAI 插件目录 `data/plugins/` 下新建或放置 `reminder_plugin` 文件夹。
2. 将本插件所有代码（包含 `main.py`, `schema.json`, `manifest.json`, `index.html` 等）解压放入该目录。
3. 通过 KiraAI 框架管理后台（或控制台）加载本插件。
4. 可在插件设置或 `schema.json` GUI 配置面修改如下进阶选项：
   - `admin_users`: 填入您的超级管理员账号/QQ（支持列表形式一行一个），拥有全域查询和调度管控特权。
   - `web_port`: Web 大屏服务绑定的宿主机端口号（默认为 `8080`，若被占用插件将在 8080-8089 范围内自动递增寻找可用端口）。

## 💻 快捷指令清单 (人类交互方案)

在您的终端或聊天界面发送以下极速文本指令（亦支持“-r”等别名缩写）：

- `/r help` 或 `/待办 帮助`：查看可用指令列表全貌
- `/r` 或 `/待办`：查看本账号下的私有待办列表与对应任务序号
- `/r add <YYYY-MM-DD HH:MM> <内容>`：极简添加定时提醒（例：`/r add 2026-03-25 14:00 开会`）
- `/r rm <序号>` / `/r pause <序号>` / `/r resume <序号>`：删除 / 暂停 / 恢复 指定序号的提醒
- `/r view <序号>`：查看某条提醒的多维详细参数配置

> 💡 **管理员专属超维指令**：配置过的超管可使用 `/r all` 列举平台全体成员待办，或发送 `/r view @用户名` 定向透视某位用户的待办矩阵。

## 🤖 开放给大模型 (LLM) 的全自动工具链

您的智能体 (AI Persona) 可通过以下无缝工具深度接管用户提醒池：
- `set_reminder`：注册提醒（涵盖单次/周期/动态随机/行为推测指令等多元化参数）。
- `list_reminders`：获取对话栈当前的活跃提醒数据与属性标签。
- `delete_reminder` / `confirm_delete_reminder`：执行常态/重要的安全双模删除流程（带 Token 控制拦截）。
- `edit_reminder`：覆盖修改现存提醒（静默刷新调度树）。
- `mark_reminder_important` / `unmark_reminder_important`：施加 / 剥离防删高敏判定保护。
- `pause_reminder` / `resume_reminder`：冻结或重新唤醒休眠期的指定任务实体。

## 📝 框架与存储约定
此插件无缝向后兼容，全面摒弃跨组件侵入。所有独立运行时生成的数据均实施原子排他写锁，可靠落盘于：
`[主目录]/data/plugin_data/reminder_plugin/reminders.json`
